# Contributing to Quorim

Thank you for your interest in contributing to Quorim.
We welcome contributions that improve the project while maintaining its quality, consistency, and vision.

---

## 📌 Guidelines

Before contributing, please ensure that:
- Your code is clean, readable, and follows consistent formatting
- Changes are purposeful and aligned with the project's goals
- You have tested your changes before submitting

---

## 🔧 How to Contribute

1. Fork the repository
2. Create a new branch for your changes  
   (`feature/your-feature-name` or `fix/your-fix-name`)
3. Make your changes
4. Commit with clear and descriptive messages
5. Open a Pull Request

---

## 🧾 Code Standards

- Follow existing project structure and patterns
- Avoid unnecessary dependencies
- Keep functions and modules focused and minimal
- Write meaningful variable and function names

---

---

## 🚫 Restrictions

The following contributions may be rejected:
- Low-quality or untested code
- Breaking changes without proper justification
- Code that conflicts with the project's direction
- Malicious or abusive content

---

## 📢 Notes

- Maintainers reserve the right to accept or reject any contribution
- Large changes should be discussed before implementation

---

## 🤝 Conduct

Be respectful and professional when contributing. Toxic or disruptive behavior will not be tolerated.
